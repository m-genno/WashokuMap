和食店に特化したインバウンドを狙ったWebアプリケーションの開発を考えている。

# インバウンド客に特化
- インバウンド客はアプリをインストールするのを嫌がる傾向があることを踏まえ、Webアプリとする。
- ユーザ登録などの面倒な操作も足かせとなるため、登録しなくても利用可とする。

# 和食に特化
- 食べログや一休といった他サイトに登録されていない店舗を主ターゲットにする想定。
- 非掲載店舗への営業は人手で行っていく。
- 他サイトに登録済みの店舗でも検索および予約可能とする。

# UI
- 店名やジャンルなど自由な言葉で和食店を検索
- 検索結果が地図と一覧に表示される
- 地図もしくは一覧から店舗を選ぶと詳細が表示される。ここから店舗の詳細情報、電話やWeb予約などが可能
- ユーザの検索履歴やお気に入りを保存可能としたい

# 相談したいこと
- ユーザ登録不要だけど、予約時に毎回ユーザ情報の入力は不便なのでどこかに保存したい。
　その場合はブラウザ側データになるのか？
　それともGoogleと連携などでユーザ登録の手順を不要にして、こちらのサーバ側でユーザごとのデータを持つことは可能？
- Web予約はどのように構築すればよいか。

# 使い方(ローカル開発)

cp .env.example .env.local
npm run db:up        # DB起動
npm run dev          # アプリ起動

---

ローカルでの動かし方

npm run db:up            # DB起動(スキーマ自動適用済み)
# サンプル投入(任意):
docker exec -i washoku-db psql -U postgres -d washokumap -f - < db/seed/sample_restaurants.sql
npm run dev              # http://localhost:3000 → 検索フォームから /search

---

現在DBには合計 212件(バルク200 + サンプル5 + 既存テスト7)が入っています。データは Docker ボリュームに永続化されているので、docker compose up で再開すればそのまま使えます(消えた場合は npm run db:up 後に npm run db:seed)。

▎ 補足:管理画面の一覧・検索・絞り込みや公開検索の「件数が多い状態」での挙動確認に使えます。バルク分だけ消したいときは上記の delete … like 'bbbb0000-%' でOKです。
