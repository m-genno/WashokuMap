# WashokuMap コスト見積もり

> Status: Draft / 最終更新: 2026-06-21
> 関連: [`architecture.md`](./architecture.md) / [`data-model.md`](./data-model.md)

金額は **2026年時点の概算レンジ**。為替・各社の料金改定で変動する。
最大の変動要因は **①開発をどうするか ②Google Mapsの利用量 ③予約デスクの人件費** の3つ。

---

## 1. 初期コスト(一度きり)

| 項目 | 概算 | 備考 |
|---|---|---|
| **開発費** | **0〜数百万円** | 最大の変動要因(下記 §1.1) |
| ドメイン取得 | 1,500〜3,000円/年 | `.com` / `.jp` |
| Apple Developer Program | 約15,000円/年 | **Sign in with Apple に必須**。毎年更新 |
| プライバシーポリシー/利用規約 | 0〜20万円 | DIYなら0円、APPI/GDPR対応を弁護士監修なら5〜20万円 |
| ロゴ/UIデザイン | 0〜30万円 | 内製なら0円 |
| 各種APIの初期設定 | 0円 | Google / Apple / 翻訳等の登録自体は無料 |

### 1.1 開発費の3シナリオ
| シナリオ | 概算 | 補足 |
|---|---|---|
| 完全内製(自分で開発) | 現金支出ゼロ | 時間コストのみ |
| フリーランス/小規模外注でMVP | 約100万〜500万円 | |
| 制作会社にフル外注 | 500万〜1,000万円以上 | 全機能込みだと中〜上のレンジになりやすい |

> 機能(検索・地図・予約・口コミ・管理画面・多言語・翻訳連携)を全部入れると工数は小さくない。

## 2. ランニングコスト(月次)

### 2.1 インフラ/SaaS — 小規模ローンチ時(低トラフィック想定)

| 項目 | サービス例 | 月額概算 | スケール時の注意 |
|---|---|---|---|
| ホスティング | Vercel | 0〜3,000円 | Hobbyは無料、本番はProで約3,000円〜 |
| データベース | Supabase / Neon | 0〜4,000円 | 無料枠で開始可、Proで約3,800円 |
| **地図** | Google Maps | **0〜5,000円** | **最大の変動費(§2.2)** |
| 翻訳 | DeepL API | 0〜3,000円 | 無料枠50万字/月＋**訳文キャッシュ**でほぼ0に抑制可 |
| メール送信 | Resend / SendGrid | 0〜3,000円 | 数千通/月までは無料枠内 |
| SMS送信(任意) | Twilio | 従量(約8〜10円/通) | 予約確定通知に使うなら件数比例。メールのみなら0円 |
| 認証 | Auth.js(自前) | 0円 | Clerk等のSaaSを使うなら3,800円〜 |
| 監視/エラー追跡 | Sentry等 | 0〜3,000円 | 無料枠あり |
| **インフラ小計** | | **約3,000〜20,000円/月** | 立ち上げ期の目安 |

### 2.2 地図コスト(要注意の変動費)
Google Maps は **地図表示(Map Load)とジオコーディングが従量課金**で、トラフィック増で一気に効く。
- 地図表示: 無料枠超過後 約 **$7 / 1,000回**。月10万回表示で **約$700(約10万円)**。
- ジオコーディング: 約 $5 / 1,000回(店舗登録時のみなので少額)。
- **対策**:
  1. 地図の不要な再読込を抑える実装。
  2. 店舗の緯度経度はDBにキャッシュし、都度ジオコーディングしない。
  3. コストが膨らむなら **Mapbox**(無料枠が大きく単価も安い)へ切替を検討。

### 2.3 予約デスクの人件費(最大の隠れコスト)
本アプリは「人手で予約仲介・店舗開拓」する設計のため、**運用人件費が実質的な主コスト**。
- 立ち上げ期: 創業者自身が対応 → 現金支出は実質0円(時間コスト)。
- 拡大期: パート/業務委託で対応 → 稼働量に応じて月数万〜数十万円。
- 予約件数に比例して増えるため、**将来は一部自動化(外部予約システム連携、定型返信)で逓減**させる設計が重要。

## 3. まとめ

| | 小さく始める場合 | 外注フルローンチの場合 |
|---|---|---|
| **初期コスト** | 約2〜5万円(ドメイン+Apple+規約DIY)＋自分の開発工数 | 約500万〜1,000万円超 |
| **月次ランニング** | インフラ 約3,000〜20,000円 ＋ 人件費(当初は自分) | 上記＋人件費・保守費 |

**現実的な始め方**: まず内製 or 小規模外注でMVPを作り、無料枠中心で月1〜2万円のインフラ費＋自分の稼働で運用開始 → ユーザー増加後に地図・翻訳・人件費を最適化する立ち上げが、資金リスクを最小化できる。

## 4. 試算を精緻化するための前提(要確認)
以下が決まると、地図・翻訳・通知のコストを具体的に試算できる。
- 初期の登録店舗数(ジオコーディング回数に影響)
- 月間ユーザー数 / 想定セッション数(地図表示回数に直結 = 最大の変動費)
- 予約の月間件数(メール/SMS通知・予約デスク人件費に影響)
- SMS通知を使うか(メールのみならSMS費はゼロ)
- 開発の体制(内製 / 外注)
